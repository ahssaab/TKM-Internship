## Documentation
Project OverviewThis project features a 4-Bit Binary Coded Decimal (BCD) Adder implemented in Verilog and verified using a SystemVerilog testbench. Developed and simulated on the EDA Playground platform, this project demonstrates a complete workflow for designing and validating digital circuits.Hardware Architecture & FunctionalityThe core system computes the sum of two 4-bit BCD numbers along with an incoming carry signal (cin).Because standard binary addition can yield results greater than 9—which exceeds valid BCD formatting—the design integrates dedicated correction logic. If the initial sum exceeds 9 or generates an intermediate carry, the hardware automatically adds a correction factor of 6 ($0110_2$). This ensures the final output delivers a valid BCD sum and an accurate carry-out (cout).Verification SetupTo align with industry-standard verification practices, the testbench does not connect directly to the hardware design under test (DUT). Instead, it communicates via a SystemVerilog Interface. This interface encapsulates all input signals (A, B, cin) and output signals (Sum, cout) into a single, organized channel, mimicking real-world verification environments.Simulation & Waveform ResultsThe testbench evaluates the adder's logic across three distinct operational scenarios:Standard Addition: Adding 4 and 4 yields a valid BCD result of 8 with no carry-out. In this case, the correction logic remains inactive.BCD Correction Enabled: Adding 7 and 6 results in an uncorrected binary value of 13. Recognizing this as an invalid BCD state, the detection circuit triggers a correction, successfully outputting a BCD sum of 3 and a carry-out of 1.Maximum Bounary Check: To test worst-case conditions, 9, 9, and a carry-in of 1 are added together.
